Action Diffusion Modelson Real Estate House Tours

Can an action-only diffusion model interpolate a full trajectory between a start and a goal image, and beat a video-diffusion baseline at it?

Introduction

Humans navigate the world from an ego-centric view; we can make every decision and plan we need from the visual stimulus of our eyes alone. For computers, this kind of ego-centric navigation planning is much harder, and many modern action planning results sidestep the problem by relying on top-down or external views [1] [2]. In this work, we investigate whether we can instead generate action sequences to a goal position given only start and finish images from a first-person viewpoint.

Concretely, we train conditional action-diffusion models on the RealEstate10K dataset [3], a collection of egocentric house tour clips. Under two settings, open and closed conditionings, each model predicts fixed trajectory length action sequences at a 4 Hz frequency (for 2, 4, or 8 seconds). In the open conditioning setting, each model is given the (embeddings of) the start frame for a clip, and is tasked with generating the rest of the trajectory. In the closed conditioning setting, each model is given both the start and end frame for a clip, and is tasked with interpolating the trajectory. We compare this model with a baseline built on a diffusion forcing transformer [4] The question we ask is: can a action-only diffusion model outperform a video-diffusion model for a navigation task?

Both models lean on the diffusion paradigm to sample from the distribution of possible paths between the start and goal frames as there can be multiple correct action sequences from point A to point B. The interesting question is what each model is forced to learn along the way. By directly generating actions, we hypothesize that the action-diffusion model can decouple motion dynamics from image synthesis, producing trajectories that are more consistent and physically executable. The video diffusion model, on the other hand, has to commit to a full visual rollout, which leaves room for trajectories that look plausible frame-to-frame but are dynamically infeasible when you try to extract the underlying motion.

This project targets a specific gap in the literature. Diffusion Policy [1] iteratively diffuses action chunks inside a closed control loop, and existing inverse dynamics models can recover action deltas between consecutive frames, but both of these approaches rely on temporal supervision and feedback. We consider a new question: how to interpolate a full action sequence given only a ground truth start and end image.
Here, actions are represented as twists where a=(v,ω)∈ℝ6.

Background and Related Work


Diffusion Models. Introduced in results like [5], a denoising diffusion model iteratively denoises a sample from Gaussian noise to generate samples from the data distribution p(x0). It defines a forward process where a sample xk=αkx0+σkϵ is the noised value of the initial sample x0 after the cumulative Gaussian noise ε∼𝒩(0,I) is applied. The model then learns the reverse of this process via a score function sθ(xk,k)≈∇logpk(xk) which is equivalent via an affine transformation to learning the noise function εθ(xk,k) through the loss function ℒ(θ)=‖ϵθ(xk,k)−ϵ‖2
By learning this, the model must learn the original data distribution in order to determine how much noise was applied to get a certain sample.

These vanilla diffusion models can generate samples from a given distribution, yet often we want to condition the generation on some prompt. To do this, Classifier-Free Guidance (CFG) [6] trains both an unconditional model over the distribution p(x0) and a conditional model that approximates the distribution p(x0|c). Like in standard diffusion, these models are parameterized through the noise function εθ(xk,k,c), where the unconditional model simply adds a null token as the conditioning to become εθ(xk,k)=εθ(xk,k,c=∅). On sampling, a guidance scale w is introduced so that ε^k=(1+w)εθ(xk,k,c)−wεθ(xk,k). Thus, w=0 becomes standard conditioning, and any higher w>0 pushes the sample more in the direction of the conditional distribution.

In this blog, we use CFG during sampling for both the video and action diffusion models.

Action Diffusion Models. Following the diffusion model paradigm, [1] showed that one can iteratively denoise action sequences to ultimately generate an action plan distributed according to some imitation dataset. Concretely, they predict the conditional distribution p(At|Ot) and use it in a closed loop where they sample action chunks, execute them, update observations, and repeat. We use a similar idea of action denoising, yet we use a one-shot method where we generate the entire action sequence at the beginning instead of replanning during execution.

Diffusion Forcing Transformer (DFoT). Conditional video generation traditionally applies a constant noise level to each frame in the video sequence during denoising. However, Diffusion Forcing and Diffusion Forcing Transformers [4] [7] change this paradigm by applying independently random levels of noise to each frame. This allows us to view the noise as a form of temporal masking and therefore generate multiple types of videos, from interpolating between ground truth frames to generating open-horizon video sequences. This project will use DFoT as the baseline representation of state-of-the-art video generation methods for action planning.

Image Embeddings. A core issue in robotics is the lack of scalable and diverse action data. Inverse dynamics models provide a solution, by acting as a bridge between video data and action sequences. Because raw image observations are high-dimensional, and may encode a lot of information that is irrelevant for action inference, a common approach is to use a pre-trained visual encoder as a backbone for the inverse dynamics model [1]. Visual encoders such as DINO and CroCo learn feature representations from images and videos using self-supervised learning, and have shown success in many downstream computer vision tasks [1]. In particular, CroCo is trained using a Cross-view Completion objective, where the model reconstructs masked regions of one view using information from the second view of the scene [11]. Because the masked regions can usually be recovered by reasoning about scene geometry and the relative camera pose, the pretraining objective forces the encoder to learn features that capture spatial relationships between the views rather than just semantics. Pretraining on millions of real-world image pairs (on indoor and outdoor scenes) makes CroCo features well-suited as a backbone for downstream geometric tasks.

Methods

Action Representations. For action representation, we converted consecutive camera poses into relative special euclidean group, SE(3) motions. An SE(3) transformation consists of a rotation R and a translation t . Each frame provides a 3×4 camera extrinsic matrix E=[R|t], and for every adjacent frame pair we computed the relative transform Erel=Et+1Et-1 using the analytic SE(3) inverse. This relative pose represents the motion of the camera between consecutive frames.

To obtain a compact and regression-friendly action target, we mapped each relative transform into its Lie algebra representation using the SE(3) logarithm. This produces a 6D twist vector a=(v,ω), where v∈ℝ3 represents translational motion and ω∈ℝ3 represents rotational motion in axis-angle form. The rotational component was extracted from the relative rotation matrix using the SO(3) log map, while the translational component was recovered using the inverse left Jacobian of SO(3), which accounts for coupling between rotation and translation.

Inverse Dynamics Models. We trained both an MLP and a Transformer head on the CrocoV2 features. CrocoV2 is specifically suited for downstream pose estimation tasks like Dust3R [8], making it a reasonable choice for a pre-training task for an inverse dynamics model. We tried to train the IDM with a DINOv2 embedding [12] and also a pure CNN from pixel-space like in [9], but the DINOv2's invariance to translations filtered out any pose-relevant information and the pixel-space IDM was impractical due to compute limitations.

Our global action frequency is set at 4 Hz, so thus the IDM must take two consecutive frame embeddings that occur 0.25s apart and return the inferred action. We compute the ground truth action delta ai using the method from above and then predict the inferred action a^=f(zi,zi+1) using Smooth-L1 loss. To reduce magnitude bias, each action is normalized per dimension then denormalized after the forward pass. We use an Adam optimizer with a learning rate of 10−4 and linear warmup over 1000 steps. The batch size is 64 and we train for 200 epochs.

The CrocoV2 embeddings in the training set are of size [N,196,768] where N≈50000 is the number of image embedding pairs, so to process these we introduce two IDM heads. The first takes a mean pool of the train set to collapse it to size [N,768]. Then, fmlp(zi,zi+1)=g(zi,zi+1,zi+1−zi) where g is a neural network of two linear layers with dropout, layernorm, and a GELU activation function. The second head performs self-attention over (CLS,zi,zi+1), where CLS is a learnable token. Learnable 1D positional embeddings (one per patch, shared across the two images) and a learnable image-identity embedding (one vector per image) are added. Four pre-norm Transformer encoder blocks (d=768, n_heads=12, mlp_ratio=4) run over the resulting length(1+2N=393,768) sequence. Two linear heads from the final CLS embedding produce a^∈ℝ6.

Baseline Models (DFOT + IDM). For our video-based baseline, we used a Diffusion Forcing Transformer (DFoT), with actions derived with our CroCo inverse dynamics model. We used pretrained DFoT models from the authors, which were trained on the same RealEstate10K dataset. DFoT allows us to generate interpolated videos between a start and end frame as well as open-ended videos solely given a start frame. After generating the video frames, we run the IDM to generate the inferred actions.

Action Diffusion Model

Figure 1: Overview of the action-conditioned diffusion model architecture. The MLP head (left) meanpools the image embeddings, concatenates them with the actions, and then feeds through a series of fully connected layers. The transformer head (right) feeds the actions through a MLP and then executes a series of self and cross-attention blocks with the image embeddings.

We learn a denoising action diffusion model where, given embeddings of a start frame z0 and an end frame zt , we sample from the learned action sequence distribution p ( a1 , … , at | z0 , zt ) . Like Diffusion Policy [1], our model denoises action sequences, yet our model has no control loop and is not conditioned on a per-timestep visual observation. We can also replace a null token with the end frame to sample from an open-horizon distribution p ( a1 , … , at | z0 ) = p ( a1 , … , at | z0 , ∅ ) . We use a simple DDPM here with CFG for conditioning, a sinusoidal time embedding, and two separate head architectures, much like the IDM. Importantly, on sampling, the model clips the intermediate x values so they do not rise to extreme values if the score function falls off the manifold.

The MLP head again flattens the CrocoV2 tokens so that flat ( z0 ) , flat ( zt ) ∈ ℝ768 . It then concatenates the two images and runs them through a 2-layer MLP to get a conditioning vector. The diffusion head is then a 2-layer MLP with LayerNorm and GELU activation. The transformer head concatenates the image embeddings along the sequence dimension, optionally adding a learned “image-of-origin” embedding to differentiate between the start and finish. The diffusion head then takes the n action tokens, adds positional and time embeddings, and then performs self-attention on the action tokens and cross-attention from the action tokens to the image tokens, repeating L = 6 times.

Experiments and Results

Experimental Setup. We evaluated both a baseline video generation pipeline and our proposed action diffusion framework on 2, 4, and 8 second video trajectories, each at 4 frames-per-seconds (fps) and corresponding to 8, 16, and 32 frames respectively. We evaluate both standard and interpolated video generation settings. In the standard setting, both the DFoT video diffusion model and our custom action diffusion model were conditioned on only the initial frame. In the interpolated setting, both models were conditioned on the first and last frames of a trajectory and tasked with generating intermediate frames.
All experiments use the Real-Estate-10K-Mini split from DFoT. For each test clip we take the first window of the requested length, condition on the ground-truth embeddings of its endpoint frames, and (since the action sampler is stochastic) average over eight sampled action sequences per setting. We report action MSE (per-element error of the predicted 6D twists against ground truth) and, after integrating the predicted twists into one start-to-finish SE(3) transform, the rotation error (residual angle, degrees) and translation error (residual L2, meters) of the final pose. For the action model we also sweep the guidance scale w over {0, 1, 2} and the clamp on the predicted clean sample x0 over {1, 3, 5, 8}.

Sampler settings. Across the sweep (Figure 2), a tight x0-clamp is best: loosening it from 1 toward 8 monotonically worsens every metric in every setting. Tight clamping prevents the denoising process from falling off the data manifold and producing extreme values; the true actions seem to have a thin ridge where the diffusion model produces an accurate distribution. Similarly, lower guidance values help the models as too much guidance can push the action generation towards the mode of the data, hurting its MSE. RealEstate10K has a narrow distribution of actions, forcing us to take measures to prevent the diffusion model from producing too extreme generated values. The MLP head especially drops off in performance with longer horizons, showing how it fails to scale with longer action sequences. The transformer’s attention mechanisms allow it to retain cross-sequence relationships with respect to the conditioning, scaling it better with longer horizons. We thus pin the sampler at w=1, x0-clip = 1 for the comparisons below.

Each 2×3 grid is indexed by conditioning mode (rows: open / start-only and closed / start+end) and trajectory length (columns: 2, 4, 8 s). The six curves per panel are the two heads (orange = MLP, teal = Transformer) crossed with three guidance scales (solid w=0, dashed w=1, dotted w=2); lower is better.


Where does the error lie? The per-component MSE (Figure 3) shows almost all error sits in forward translation vz, the direction of the most variance and thus the one that may differ the most from the ground truth action. Pitch ωx and roll ωz essentially zero because house-tour footage is almost all pan ωy; those near-zero components are low-variance and the model can pin them near zero.

Figure 3: Per-component action MSE at the chosen sampling setting (w=0, x0-clip = 1). Layout matches Figure 2. The six groups per panel are the twist components, linear velocity (vx,vy,vz) and angular velocity (ωx,ωy,ωz), with MLP (orange) and Transformer (teal) bars on a log scale. Error is dominated by forward translation vz at every horizon.


Conditioning on the goal frame. The effect is small and inconsistent. The goal frame never makes a decisive difference and at the longest horizon looks essentially ignored, which is itself a limit of the start-and-goal framing. Future work could involve enforcing more weight on the end goal condition.

Action diffusion vs the video-diffusion baseline. As seen in Figure 4, across all trajectory lengths, the DFoT + CroCo-IDM baseline still produced more accurate trajectories than our action diffusion models, by roughly 2× on pose error with consistently lower action MSE. While these results suggest that frame-conditioned inverse dynamics pipelines remain highly effective for navigation planning, the comparison is preliminary since the DFoT baseline used large pretrained checkpoints whereas our action diffusion models were trained from scratch under a limited compute budget.

What can the diffusion models help with?

Figure 5: Visualizing the action trajectories in a 2D space shows that the diffusion model overshoots the ground truth and has high variance across samples while the video baseline sticks closer to the ground truth action.

It seems that the diffusion model does perform more poorly than the video diffusion model when we try to recreate the ground truth. However, as seen in Figure 5, they embody different patterns. The baseline generates an action that is more similar to the ground truth, as shown in the above results. The diffusion model, in contrast, generates actions that are too large and highly variant. According to our metrics, the diffusion model performs much more poorly. However, if the goal is to rollout multiple possible and feasible trajectories, the diffusion model could show more promise, as seen in [1]. This is a possible avenue for further inquiry.

Implications and limitations

Implications: On this data the action-only model did not win: the DFoT-plus-IDM baseline produced more accurate trajectories at every horizon. We are cautious about reading too much into that, because the comparison is not clean (DFoT is a large model pretrained on the full dataset; our action model is small, trained from scratch on the Mini split, and compute-limited). But three things hold up regardless. First, frame-conditioned inverse-dynamics models are stronger than we gave them credit for, and since video diffusion is improving faster than action-specific generative models, riding that curve through a good IDM may be a more scalable path than building bespoke action models. Second, the failure pattern of the action model is informative on its own: error compounds with horizon, the goal frame is effectively ignored at the longest horizons, and the sampler is fragile off-manifold. That pattern is consistent with start-and-goal conditioning being too sparse a signal compared with the dense intermediate-frame scaffolding the IDM gets from DFoT, which points toward hybrid approaches that use video diffusion for a few visual anchors and condition action diffusion on those rather than on endpoints alone. Third, the diffusion model does embody higher variance, potentially showing promise for modeling multimodal or more wide distributions of navigation sequences. The original hypothesis, that directly modeling the action distribution decouples motion dynamics from image synthesis and dodges the dynamically-infeasible-rollout failure mode, is not refuted here, but our experiments do not support it either; testing it properly needs the apples-to-apples training run and the executability-based evaluation we describe below.

Limitations. A few caveats on what we found. First, our evaluation metrics are imperfect proxies. Action MSE doesn't capture whether the generated trajectory is actually executable; a simulator rollout that executes the predicted twists and measures final pose error would be more meaningful. Second, we never do closed-loop testing: we predict trajectories one-shot and never execute and replan, but real navigation isn't like that. Third, RealEstate10K is camera tours, not robot navigation, so the action distribution reflects what a human videographer films rather than what a robot policy would produce. Finally, IDM quality bounds our baseline. The DFoT pipeline depends on the IDM being good enough to recover actions from generated frames, so if the IDM is the bottleneck, we're not really comparing action diffusion to video diffusion. We're comparing it to the IDM's ceiling.

Future work. Several of our choices were driven by compute rather than design. 4 Hz is fine for RealEstate10K but low for real robot control, where locomotion policies run at 10 to 500 Hz. We capped trajectories at short horizons because longer sequences blow up memory in the diffusion transformer, trained on RealEstate10K-Mini instead of the full dataset, and never got to sweep over conditioning strategies, action representations, or guidance scales. With more compute, the experiments we'd most want to run are: (1) a backbone scale ablation swapping in CroCo v2 Large to test whether richer features close the gap; (2) a conditioning ablation comparing start-only, start+end, and start+waypoints+end, since intermediate waypoints might give action diffusion the scaffolding DFoT gets from generated frames; (3) a stronger IDM baseline (full CroCo v2 or a Dust3R-style direct pose regression model) to disentangle whether our results are about action vs video diffusion or just about IDM quality; and (4) simply training the action model longer and on the full dataset, since the baseline's edge may reflect that data-and-compute asymmetry rather than the paradigm. We had no compute budget left to run any of these for this report.

Beyond compute, the most interesting direction is moving to simulation: train on ego-view data from a robot in Isaac Sim or Habitat and test whether the diffused action sequences are actually executable on a real platform. Even without a real robot, closed-loop rollout in sim would give a much more meaningful eval than action MSE and let us actually test the "dynamically feasible" claim from our intro. Other natural extensions include hierarchical diffusion (coarse waypoints first, then fine-grained actions, mirroring how humans plan navigation), multi-modal conditioning with language goals or top-down maps, and connecting more directly to the world models literature, since action-diffusion vs video-diffusion is really a question of planning in action space vs observation space.
