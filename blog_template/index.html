Introduction

Humans are used to seeing the world through an ego-centric view; we can make all the decision and plans we need from the visual stimulus of our eyes alone. When placed in a room, a human can instantly determine a trajectory to the desired goal location. This ego-centric navigation planning for computers, on the other hand, is a difficult task; many modern action planning results rely on top or side view cameras [1] [2]. Thus, we want to see if we could generate action sequences to a goal position given only start and finish images.

Concretely, we train a conditional action-diffusion model on the RealEstate10K dataset [3], a collection of ego-view house tour clips. Given the (embeddings of) start and goal images from a certain clip, we diffuse action sequences of variable lengths (FIX - IS TS TRUE) at a 4 Hz frequency. We compare this model with a baseline diffusion forcing transformer [4] that infers an interpolated sequence of frames in between the start and goal frame. Given this inferred sequence, we run an inverse diffusion model on consecutive frames to generate the action sequence. We thus ask the question: can a action-only diffusion model outperform a video-diffusion model for a navigation task?

Both models use the diffusion paradigm to generate an action sequence sampled from the distribution of possible paths from the start to finish, a necessary task given that there is never one unique action sequence from point A to point B. Instead, we want to sample from the distribution of all possible action sequences. However, by directly generating action sequences, we hypothesize that the generative model can decouple the action dynamics from the image generation, leading to more consistent and executable trajectories. In contrast, we wonder if the video generative model could generate visually plausable but dynamically infeasible trajectories.

This project targets a specific gap in the literature. Diffusion Policy [1] uses a control loop to iteratively diffuse action chunks. Existing inverse dynamics models can also infer action deltas between consecutive frames. However, we ask the specific question about interpolating action sequences given a ground truth start and end image.

Background and Related Work


Diffusion Models. Introduced in results like [5], a denoising diffusion model iteratively denoises a sample from Gaussian noise to generate samples from the data distribution p(x0). It defines a forward process where a sample xk=αkx0+σkϵ is the noised value of the initial sample x0 after the cumulative Gaussian noise ε∼𝒩(0,I) is applied. The model then learns the reverse of this process via a score function sθ(xk,k)≈∇logpk(xk) which is equivalent via an affine transformation to learning the noise function εθ(xk,k). By learning this, the model must learn the original data distribution in order to determine how much noise was applied to get a certain sample.

These vanilla diffusion models can generate samples from a given distribution, yet often we want to condition the generation on some prompt. To do this, Classifier-Free Guidance (CFG) [6] trains both an unconditional model over the distribution p(x0) and a conditional model that approximates the distribution p(x0|c). Like in standard diffusion, these models are parameterized through the noise function εθ(xk,k,c), where the unconditional model simply adds a null token as the conditioning to become εθ(xk,k)=εθ(xk,k,c=∅). On sampling, a guidance scale w is introduced so that ε^k=(1+w)εθ(xk,k,c)−wεθ(xk,k). Thus, w=0 becaomes standard conditioning, and any higher w>0 pushes the sample more in the direction of the conditional distribution.

In this blog, we use CFG for both the generation of video frames as well as the generation of action sequences for the downstream navigation task.

Action Diffusion Models. Following the diffusion model paradigm, [1] showed that one can iteratively denoise action sequences to ultimately generate a action plan distributed according to some imitation dataset. Concretely, they predict the conditional distribition p(At|Ot) and use it in a closed loop where they then sample an action chunk, execute part of it, recollect the observation buffer, and repeat. We use a similar idea of action denoising, yet we use a one-shot method where we generate the entire action sequence at the beginning instead of using the control loop.

Diffusion Forcing Transformer (DFoT). Conditional video generation originally applied a constant noise level to each frame in the video sequence to then denoise. However, Diffusion Forcing and Diffusion Forcing Transformers [4] [7] change this paradigm by applying independently random levels of noise to each frame. This allows us to view the noise as a form of masking and therefore generate multiple types of videos, from interpolating between ground truth frames to generating open-horizon video sequences. This project will use DFoT for video generation conditioned on both the start and end frames (interpolation) and just the start frames to get a possible navigation sequence.

Inverse Dynamic Models. A core issue in robotics is the lack of scalable and diverse action data. Inverse dynamics models provide a solution, by acting as a bridge between video data and action sequences. Because raw image observations are high-dimensional, and may encode a lot of information that is irrelevant for action inference, a common approach is to use a pre-trained visual encoder as a backbone for the inverse dynamics model (CITE). Visual encoders such as DINO and CroCo learn feature representations from images and videos using self-supervised learning, and have shown success in many downstream computer vision tasks (CITE DINO). In particular, CroCo is trained using a Cross-view Completion objective, where the model is given a sequence of frames with one frame missing, and must predict the missing frame, and thus are a promising backbone for our video-based inverse dynamics modeling.

Methods

Action Representations. For action represetation, we converted consecutive camera poses into relative special euclidean group, SE(3) motions.

An SE(3) transformation consists of:

  • Rotation: how the object turns in 3D space
  • Translation: how the object moves in x, y, z
Each frame provides a 3×4 camera extrinsic matrix E=[R|t], and for every adjacent frame pair we computed the relative transform Erel=Et+1Et-1 using the analytic SE(3) inverse. This relative pose represents the motion of the camera between consecutive frames.

To obtain a compact and regression-friendly action target, we mapped each relative transform into its Lie algebra representation using the SE(3) logarithm. This produces a 6D twist vector a=(v,ω), where v∈ℝ3 represents translational motion and ω∈ℝ3 represents rotational motion in axis-angle form. The rotational component was extracted from the relative rotation matrix using the SO(3) log map, while the translational component was recovered using the inverse left Jacobian of SO(3), which accounts for coupling between rotation and translation.



Inverse Dynamics Models. We trained both an MLP (multi-layer perceptron) and a Transformer head on the DINO features. However, we found that the DINO features were not well-suited for our task, as they were trained on a different domain and did not capture the relevant information for action inference in our setting. (include loss on MLP and Transformer head DINO) We also tried a simple MLP architecture without a backbone, as described in (cnn idm citation). This model performed the worse, and was unable to converge given our training parameters. For our final inverse dynamics model, we used a CroCo backbone, which is trained on video data and thus has more relevant features for our task. We trained both a simple MPL head and a transformer head on top of the CroCo features, and found that the transformer head performed better, likely due to the limited amount of training data we had for this task.
