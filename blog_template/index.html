Introduction

Humans navigate the world from an ego-centric view; we can make every decision and plan we need from the visual stimulus of our eyes alone. When placed in a room, a person can instantly determine a trajectory to a desired goal location. For computers, this kind of ego-centric navigation planning is much harder, and many modern action planning results sidestep the problem by relying on top-down or side-view cameras [1] [2]. We want to see whether we can instead generate action sequences to a goal position given only start and finish images.

Concretely, we train a conditional action-diffusion model on the RealEstate10K dataset [3], a collection of egocentric house tour clips. Given the (embeddings of) start and goal images from a clip, we diffuse action sequences of variable lengths (FIX - IS TS TRUE) at a 4 Hz frequency. We compare this model with a baseline built on a diffusion forcing transformer [4] which interpolates a sequence of frames between the start and goal and then runs an inverse dynamics model on consecutive frames to recover the action sequence. The question we ask is: can a action-only diffusion model outperform a video-diffusion model for a navigation task?

Both models lean on the diffusion paradigm to sample from the distribution of possible paths between the start and goal frames, which is the right framing because there is never a single correct action sequence from point A to point B. The interesting question is what each model is forced to learn along the way. By directly generating actions, we hypothesize that the action-diffusion model can decouple motion dynamics from image synthesis, producing trajectories that are more consistent and physically executable. The video diffusion model, on the other hand, has to commit to a full visual rollout, which leaves room for trajectories that look plausible frame-to-frame but are dynamically infeasible when you try to extract the underlying motion.

This project targets a specific gap in the literature. Diffusion Policy [1] iteratively diffuses action chunks inside a closed control loop, and existing inverse dynamics models can recover action deltas between consecutive frames. Neither directly addresses our question: how to interpolate a full action sequence given only a ground truth start and end image.
Here, actions are represented as twists where a=(v,ω)∈ℝ6.

Background and Related Work


Diffusion Models. Introduced in results like [5], a denoising diffusion model iteratively denoises a sample from Gaussian noise to generate samples from the data distribution p(x0). It defines a forward process where a sample xk=αkx0+σkϵ is the noised value of the initial sample x0 after the cumulative Gaussian noise ε∼𝒩(0,I) is applied. The model then learns the reverse of this process via a score function sθ(xk,k)≈∇logpk(xk) which is equivalent via an affine transformation to learning the noise function εθ(xk,k) through the loss function ℒ(θ)=‖ϵθ(xk,k)−ϵ‖2
By learning this, the model must learn the original data distribution in order to determine how much noise was applied to get a certain sample.

These vanilla diffusion models can generate samples from a given distribution, yet often we want to condition the generation on some prompt. To do this, Classifier-Free Guidance (CFG) [6] trains both an unconditional model over the distribution p(x0) and a conditional model that approximates the distribution p(x0|c). Like in standard diffusion, these models are parameterized through the noise function εθ(xk,k,c), where the unconditional model simply adds a null token as the conditioning to become εθ(xk,k)=εθ(xk,k,c=∅). On sampling, a guidance scale w is introduced so that ε^k=(1+w)εθ(xk,k,c)−wεθ(xk,k). Thus, w=0 becomes standard conditioning, and any higher w>0 pushes the sample more in the direction of the conditional distribution.

In this blog, we use CFG for both the generation of video frames as well as the generation of action sequences for the downstream navigation task.

Action Diffusion Models. Following the diffusion model paradigm, [1] showed that one can iteratively denoise action sequences to ultimately generate an action plan distributed according to some imitation dataset. Concretely, they predict the conditional distribution p(At|Ot) and use it in a closed loop where they then sample an action chunk, execute part of it, recollect the observation buffer, and repeat. We use a similar idea of action denoising, yet we use a one-shot method where we generate the entire action sequence at the beginning instead of using the control loop.

Diffusion Forcing Transformer (DFoT). Conditional video generation originally applied a constant noise level to each frame in the video sequence to then denoise. However, Diffusion Forcing and Diffusion Forcing Transformers [4] [7] change this paradigm by applying independently random levels of noise to each frame. This allows us to view the noise as a form of masking and therefore generate multiple types of videos, from interpolating between ground truth frames to generating open-horizon video sequences. This project will use DFoT for video generation conditioned on both the start and end frames (interpolation) and just the start frames to get a possible navigation sequence.

Image Embeddings. A core issue in robotics is the lack of scalable and diverse action data. Inverse dynamics models provide a solution, by acting as a bridge between video data and action sequences. Because raw image observations are high-dimensional, and may encode a lot of information that is irrelevant for action inference, a common approach is to use a pre-trained visual encoder as a backbone for the inverse dynamics model (CITE). Visual encoders such as DINO and CroCo learn feature representations from images and videos using self-supervised learning, and have shown success in many downstream computer vision tasks (CITE DINO). In particular, CroCo is trained using a Cross-view Completion objective, where the model is given a sequence of frames with one frame missing, and must predict the missing frame, and thus are a promising backbone for our video-based inverse dynamics modeling. CroCo extends masked image modeling to pairs of images depicting the same scene from different viewpoints. Given the two views, the model masks a large fraction of the first image and tasks a ViT-based encoder with reconstructing them, conditioned on the fully-visible second image through cross-attention in the decoder. Because the masked content of one view can usually be recovered from the other view only by reasoning about scene geometry and the relative camera pose, the pretraining objective forces the encoder to learn features that capture spatial relationships between the views rather than just semantics. Pretraining on millions of real-world image pairs (on indoor and outdoor scenes) makes CroCo features well-suited as a backbone for downstream geometric tasks.

Methods

Action Representations.

For action representation, we converted consecutive camera poses into relative special euclidean group, SE(3) motions. An SE(3) transformation consists of a rotation R and a translation t . Each frame provides a 3×4 camera extrinsic matrix E=[R|t], and for every adjacent frame pair we computed the relative transform Erel=Et+1Et-1 using the analytic SE(3) inverse. This relative pose represents the motion of the camera between consecutive frames.

To obtain a compact and regression-friendly action target, we mapped each relative transform into its Lie algebra representation using the SE(3) logarithm. This produces a 6D twist vector a=(v,ω), where v∈ℝ3 represents translational motion and ω∈ℝ3 represents rotational motion in axis-angle form. The rotational component was extracted from the relative rotation matrix using the SO(3) log map, while the translational component was recovered using the inverse left Jacobian of SO(3), which accounts for coupling between rotation and translation.

Inverse Dynamics Models.

We trained both an MLP and a Transformer head on the CrocoV2 features. CrocoV2 is specifically suited for downstream pose estimation tasks like Dust3R [8], making it a reasonable choice for a pre-training task for an inverse dynamics model. We tried to train the IDM with a DINOv2 embedding [11] and also a pure CNN from pixel-space like in [10], but the DINOv2's invariance to translations filtered out any pose-relevant information and the pixel-space IDM was impractical due to compute limitations. Thus, for our final inverse dynamics model, we used a CroCo backbone, which is trained on video data and thus has more relevant features for our task.

Our global action frequency is set at 4 Hz, so thus the IDM must take two consecutive frame embeddings that occur 0.25s apart and return the inferred action. We compute the ground truth action delta ai using the method from above and then predict the inferred action a^=f(zi,zi+1) using Smooth-L1 loss. To reduce magnitude bias, each action is normalized per dimension to have zero mean and standard deviation of 1 and then denormalized after the forward pass. We use an Adam optimizer with a learning rate of 10−4 and linear warmup over 1000 steps. The batch size is 64 and we train for 200 epochs.

The CrocoV2 embeddings in the training set are of size [N,196,768] where N≈50000 is the number of image embedding pairs, so to process these we introduce two IDM heads. The first takes a mean pool of the train set to collapse it to size [N,768]. Then, fmlp(zi,zi+1)=g(zi,zi+1,zi+1−zi) where g is a neural network of two linear layers with dropout, layernorm, and a GELU activation function. The second head performs self-attention over (CLS,zi,zi+1), where CLS is a learnable token. Learnable 1D positional embeddings (one per patch, shared across the two images) and a learnable image-identity embedding (one vector per image) are added. Four pre-norm Transformer encoder blocks (d=768, n_heads=12, mlp_ratio=4) run over the resulting length(1+2N=393,768) sequence. Two linear heads from the final CLS embedding produce a^∈ℝ6.

Action Diffusion Model

Figure 1: Overview of the action-conditioned diffusion model architecture. The MLP head (left) meanpools the image embeddings, concatenates them with the actions, and then feeds through a series of fully connected layers. The transformer head (right) feeds the actions through a MLP and then executes a series of self and cross-attention blocks with the image embeddings.

We learn a denoising action diffusion model where, given embeddings of a start frame z0 and an end frame zt , we sample from the learned action sequence distribution p ( a1 , … , at | z0 , zt ) . Like Diffusion Policy [1], our model denoises action sequences, yet our model has no control loop and is not conditioned on a per-timestep visual observation. We can also replace a null token with the end frame to sample from an open-horizon distribution p ( a1 , … , at | z0 ) = p ( a1 , … , at | z0 , ∅ ) . We use a simple DDPM here with CFG for conditioning, a sinusoidal time embedding, and two separate head architectures, much like the IDM.

The MLP head again flattens the CrocoV2 tokens so that flat ( z0 ) , flat ( zt ) ∈ ℝ768 . It then concatenates the two images and runs them through a 2-layer MLP to get a conditioning vector. The diffusion head is then a 2-layer MLP with LayerNorm and GELU activation.

The transformer head concatenates the image embeddings along the sequence dimension, optionally adding a learned “image-of-origin” embedding to differentiate between the start and finish. The diffusion head then takes the n action tokens, adds positional and time embeddings, and then performs self-attention on the action tokens and cross-attention from the action tokens to the image tokens, repeating L = 6 times.

Experiments and Results

Experimental Setup We evaluated both a baseline video generation pipeline and our proposed action diffusion framework on 2, 4, and 8 second video trajectories, each at 4 frames-per-seconds (fps) and corresponding to 8, 16, and 32 frames respectively. We evaluate both standard and interpolated video generation settings. In the standard setting, both the DFoT video diffusion model and our custom action diffusion model were conditioned on only the initial frame. In the interpolated setting, both models were conditioned on the first and last frames of a trajectory and tasked with generating intermediate frames.

Implications and limitations

Implications: What our comparison says about action-only vs video-based trajectory generation depends on which way the results lean. If action diffusion wins, that supports our core hypothesis: directly modeling the action distribution decouples motion dynamics from image synthesis and avoids the failure mode where visually plausible frames yield dynamically infeasible trajectories. If the two perform comparably, the more interesting reading is that frame-conditioned IDMs are stronger than we gave them credit for, and since video diffusion is improving faster than action-specific generative models, riding that curve via a good IDM may be a more scalable path than building bespoke action models. And if action diffusion underperforms, the likely explanation is that start-and-goal conditioning is too sparse compared to the dense scaffolding DFoT gives the IDM through generated intermediate frames, pointing toward hybrid approaches that use video diffusion for sparse visual anchors and condition action diffusion on those.

Limitations. A few caveats on what we found. First, our evaluation metrics are imperfect proxies. Action MSE doesn't capture whether the generated trajectory is actually executable; a simulator rollout that executes the predicted twists and measures final pose error would be more meaningful. Second, we never do closed-loop testing: we predict trajectories one-shot and never execute and replan, but real navigation isn't like that. Third, RealEstate10K is camera tours, not robot navigation, so the action distribution reflects what a human videographer films rather than what a robot policy would produce. Finally, IDM quality bounds our baseline. The DFoT pipeline depends on the IDM being good enough to recover actions from generated frames, so if the IDM is the bottleneck, we're not really comparing action diffusion to video diffusion. We're comparing it to the IDM's ceiling.

Future work. Several of our choices were driven by compute rather than design. 4 Hz is fine for RealEstate10K but low for real robot control, where locomotion policies run at 10 to 500 Hz. We capped trajectories at short horizons because longer sequences blow up memory in the diffusion transformer, trained on RealEstate10K-Mini instead of the full dataset, and never got to sweep over conditioning strategies, action representations, or guidance scales. With more compute, the experiments we'd most want to run are: (1) a backbone scale ablation swapping in CroCo v2 Large to test whether richer features close the gap; (2) a conditioning ablation comparing start-only, start+end, and start+waypoints+end, since intermediate waypoints might give action diffusion the scaffolding DFoT gets from generated frames; (3) a stronger IDM baseline (full CroCo v2 or a Dust3R-style direct pose regression model) to disentangle whether our results are about action vs video diffusion or just about IDM quality.

Beyond compute, the most interesting direction is moving to simulation: train on ego-view data from a robot in Isaac Sim or Habitat and test whether the diffused action sequences are actually executable on a real platform. Even without a real robot, closed-loop rollout in sim would give a much more meaningful eval than action MSE and let us actually test the "dynamically feasible" claim from our intro. Other natural extensions include hierarchical diffusion (coarse waypoints first, then fine-grained actions, mirroring how humans plan navigation), multi-modal conditioning with language goals or top-down maps, and connecting more directly to the world models literature, since action-diffusion vs video-diffusion is really a question of planning in action space vs observation space.
